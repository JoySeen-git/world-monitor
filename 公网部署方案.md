# World Monitor 公网部署方案

## 📦 部署目标
将 World Monitor 实时全球地缘政治态势感知仪表盘部署到公网，支持 HTTPS 访问和实时更新。

## 🌐 部署架构

```
用户 → HTTPS (443) → Nginx 反向代理 → 前端 (3000)
                          ↓
                     后端 API (3001)
                          ↓
                     WebSocket (3001/ws)
```

## 🚀 推荐部署方案

### 方案一：VPS 部署（推荐）

#### 1. 服务器要求
- **CPU**: 2 核心以上
- **内存**: 4GB 以上
- **存储**: 20GB SSD
- **带宽**: 5Mbps 以上
- **系统**: Ubuntu 22.04 LTS / Debian 11

#### 2. 推荐服务商
- **国内**（需备案）：
  - 阿里云 ECS
  - 腾讯云 CVM
  - 华为云
- **国外**（无需备案）：
  - AWS Lightsail
  - DigitalOcean Droplet
  - Vultr
  - Linode

#### 3. 部署步骤

##### 3.1 服务器准备
```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2 进程管理器
sudo npm install -g pm2

# 安装 Nginx
sudo apt install nginx -y

# 配置防火墙
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
```

##### 3.2 上传代码
```bash
# 创建应用目录
mkdir -p /var/www/world-monitor
cd /var/www/world-monitor

# 上传项目文件（使用 SCP 或 Git）
# 方式 1: SCP
scp -r ./world-monitor/* user@server:/var/www/world-monitor/

# 方式 2: Git
git clone <repository-url> .
```

##### 3.3 安装依赖
```bash
# 安装后端依赖
cd /var/www/world-monitor/backend
npm install

# 安装前端依赖
cd ../frontend
npm install

# 构建前端
npm run build
```

##### 3.4 配置环境变量
```bash
# 创建后端环境配置
cd /var/www/world-monitor/backend
cat > .env << EOF
PORT=3001
HOST=0.0.0.0
NODE_ENV=production
DB_PATH=/var/www/world-monitor/backend/data/db.json
EOF
```

##### 3.5 启动服务
```bash
# 使用 PM2 启动后端
cd /var/www/world-monitor/backend
pm2 start npm --name "world-monitor-api" -- run dev

# 使用 PM2 启动前端
cd /var/www/world-monitor/frontend
pm2 start npm --name "world-monitor-frontend" -- run dev

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
```

##### 3.6 配置 Nginx 反向代理
```bash
sudo nano /etc/nginx/sites-available/world-monitor
```

添加以下配置：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 前端
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    # 后端 API
    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    # WebSocket
    location /ws {
        proxy_pass http://localhost:3001/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/world-monitor /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

##### 3.7 配置 HTTPS（Let's Encrypt）
```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo crontab -e
# 添加：0 3 * * * certbot renew --quiet
```

### 方案二：Docker 部署

#### 1. 创建 Dockerfile

**后端 Dockerfile** (`backend/Dockerfile`):
```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --production

COPY . .

EXPOSE 3001

CMD ["npm", "run", "dev"]
```

**前端 Dockerfile** (`frontend/Dockerfile`):
```dockerfile
FROM node:20-alpine as build

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

#### 2. Docker Compose 配置

创建 `docker-compose.yml`:
```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "3001:3001"
    volumes:
      - ./backend/data:/app/data
    environment:
      - PORT=3001
      - HOST=0.0.0.0
      - NODE_ENV=production
    restart: always

  frontend:
    build: ./frontend
    ports:
      - "3000:80"
    depends_on:
      - backend
    restart: always

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - frontend
      - backend
    restart: always
```

#### 3. 部署命令
```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 🔒 安全配置

### 1. 防火墙配置
```bash
# 只开放必要端口
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
```

### 2. SSL/TLS 配置
```nginx
ssl_protocols TLSv1.2 TLSv1.3;
ssl_prefer_server_ciphers on;
ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
```

### 3. 速率限制
```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;

server {
    location /api {
        limit_req zone=one burst=20 nodelay;
        # ...
    }
}
```

## 📈 监控与维护

### 1. 日志管理
```bash
# 查看 PM2 日志
pm2 logs

# 查看 Nginx 日志
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

### 2. 自动备份
```bash
# 创建备份脚本
cat > /var/www/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
tar -czf /var/backups/world-monitor-$DATE.tar.gz /var/www/world-monitor/backend/data
find /var/backups -name "world-monitor-*.tar.gz" -mtime +7 -delete
EOF

chmod +x /var/www/backup.sh

# 添加定时任务
crontab -e
# 0 2 * * * /var/www/backup.sh
```

### 3. 健康检查
```bash
# 创建健康检查脚本
cat > /var/www/healthcheck.sh << 'EOF'
#!/bin/bash
curl -f http://localhost:3001/api/statistics || exit 1
EOF

chmod +x /var/www/healthcheck.sh
```
