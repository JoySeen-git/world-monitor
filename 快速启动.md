# 🌍 World Monitor 快速启动指南

## ✅ 系统已就绪

World Monitor（实时全球地缘政治态势感知仪表盘）已经搭建完成！

## 📍 访问地址

- **前端界面**: http://localhost:3000
- **后端 API**: http://localhost:3001
- **WebSocket**: ws://localhost:3001/ws

## 🚀 启动方式

### 方式一：使用启动脚本（推荐）

双击运行项目根目录下的 `启动.bat` 文件即可同时启动前后端服务。

### 方式二：手动启动

**1. 启动后端（终端 1）**
```bash
cd backend
npm run dev
```

**2. 启动前端（终端 2）**
```bash
cd frontend
npm run dev
```

### 已实现功能

1. **全球事件地图**
   - 实时展示全球地缘政治事件

6. **WebSocket 实时更新**
   - 数据自动推送
   - 无需手动刷新

## 🔧 配置选项

### 1. 配置 NewsAPI（可选）

编辑 `backend/.env` 文件：

```env
NEWSAPI_KEY=your_newsapi_key_here
```

获取 API Key: https://newsapi.org/

### 2. 调整更新频率

编辑 `backend/src/index.ts` 中的 cron 表达式：

```typescript
cron.schedule('*/15 * * * *', async () => {
  // 每 15 分钟更新一次
})
```

### 3. 添加更多数据源

在 `backend/src/index.ts` 中添加新的数据获取函数。

## ⚠️ 注意事项

1. **GDELT API 限流**
   - GDELT 有请求频率限制
   - 默认使用缓存数据
   - 建议不要频繁手动刷新

2. **端口占用**
   - 后端使用 3001 端口
   - 前端使用 3000 端口
   - 如有冲突请修改配置

3. **数据持久化**
   - 数据存储在 `backend/data/db.json`
   - 可手动编辑或清空数据

## 🐛 故障排查

### 前端无法访问
1. 确认前端服务已启动
2. 检查浏览器控制台错误
3. 清除浏览器缓存

### 后端无法连接
1. 检查端口 3001 是否被占用
2. 查看后端日志
3. 确认 `.env` 配置正确

### 没有数据
1. 检查是否运行了种子脚本
2. 查看 `backend/data/db.json` 是否存在
3. 等待数据自动更新

### WebSocket 未连接
1. 检查防火墙设置
2. 确认后端 WebSocket 服务正常
3. 查看浏览器控制台连接状态

## 📚 API 文档

### REST API

**获取事件列表**
```
GET /api/events?days=7&type=Conflict&country=Israel&limit=100
```

**获取新闻列表**
```
GET /api/news?days=3&category=general&limit=50
```

**获取风险指数**
```
GET /api/risk-indices
```

**获取统计数据**
```
GET /api/statistics
```

**获取热力图数据**
```
GET /api/heatmap?days=7
```

### WebSocket 消息

客户端会收到以下类型的实时更新消息：

```json
{
  "type": "events_updated",
  "count": 10
}
```

```json
{
  "type": "news_updated"
}
```

```json
{
  "type": "risk_indices_updated"
}
```
